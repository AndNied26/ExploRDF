Help

This web application is made for visualizing RDF Graphs (Resource Description Framework).
The following guideline will try to help you to get used to the application and solve some of the occurred questions.

Connecting to a RDF Triple Store

First of all you have to connect to a RDF Triple Store. Therefore you need to fill in the form at the Connect site.
You can choose between a RDF4J-Triple-Store, a Stardog-Triple-Store or a SPARQL-Endpoint to get connected to.

After the connection attempt succeeded, the connection properties are to be refreshed and shown in the right upper corner.

Searching for a term

To search the RDF triple store for a certain term you have to enter your search term in the search field on the Explore site.

You can choose between Simple and Broader Search whereas the first search method means that the result must exactly match the search term and the latter method means that the result must only contain the search term.
Therefore, Broader Search is case insensitive. However, the second search method requires much more time than the first one.

Choosing a Starting Resource

After entering a search term a table with matching resources found in the triple store is drawn.
You have to choose one of the found resources by clicking on its "link" (IRI) in the first column.

After you have chosen a resource to begin the RDF Graph exploration with, you get another table with all database information that is linked to the chosen resource.

In the bar under the information table you have the possibility to adjust some of the parameters of how the RDF data is going to be visualized. You can specify whether only the outgoing, incoming or both kind of edges of the graph nodes are to be shown in the simulation.
Furthermore you need to choose, which predicates to be visualized in the graph. Therefore you need to select a predicates list on the left side of the bar. If you want to modify the predicates list or create a new list you need to click on the Customize button.
After clicking the button a new table with all the different predicates of the connected Triple Store is drawn.

The predicates table consists of the three columns Predicate, Label and Edge. Under the Predicate column the identifier (IRI) of each predicate can be found. In both the other cells of each row a checkbox is set. If the the Label checkbox is checked the predicate of the row is going to determine the node label in the graph visualization. Only one checked Label checkbox in a predicate list is possible. If the Edge checkbox is checked the predicate is shown as an edge in the graph simulation in case the predicate is not connecting a node to a RDF literal.
By entering a list name and clicking on the Save button the predicates list will be overwritten or created anew. After saving the predicates list you will be led back to the start resource information table. By clicking the Explore button you can start the visualization.

Exploring the RDF Graph

The visualization starts with the chosen starting node in the middle of the SVG element (white zoomable area).

By hovering over the node, six different icons appear near the node, each with an other functionality.

  • By clicking on the Expand icon the next relations of the node are visualized. Only 10 of such relations of a node are shown at the same time. If there are more than ten relations, you can click on the Expand icon for exposing the next 10 relations.
  • The Offset number shows which amount of relations is currently visualized.
  • You can remove a node from the simulation by clicking the Remove icon.
  • By clicking the Info icon all information of the nodes resource are shown in an info box.
  • A node stays always at the last position it is dragged to. But by clicking on the Unpin icon the nodes position is calculated according to the other nodes in the simulation and is automatically dragged to its new position.
  • After a node appears in the simulation, it stays 10 further exploration units (clicks on nodes) on screen. To avoid the removal of certain nodes, they can be marked manually. By clicking on the Mark icon the node remains in the simulation until its end or after undoing the mark (clicking the icon another time).
